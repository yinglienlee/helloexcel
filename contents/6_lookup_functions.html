🔍 查詢與參照函數

精通數據查找的關鍵技能

1查詢函數概述與重要性

查詢與參照函數是 Excel 中最強大且最常用的工具之一。在實務工作中,我們經常需要從大型數據表中快速找到特定資訊,而查詢函數就是解決這個問題的關鍵。

📊 實務應用場景

  • 工業工程 (IE): 根據產品編號查找標準成本、製造時間、庫存量
  • 經濟學: 根據國家代碼查找 GDP、失業率、通膨率等經濟指標
  • 人力資源: 根據員工編號查找薪資、部門、職位資訊
  • 銷售管理: 根據客戶代碼查找聯絡資訊、交易記錄
  • 財務會計: 根據科目代碼查找會計科目名稱、預算金額
💡 為什麼需要學習查詢函數?
想像你有一份包含 10,000 筆產品資料的清單,客戶詢問某個產品的價格,你不可能手動翻找。查詢函數可以在不到一秒的時間內找到答案,並且當資料更新時,結果會自動更新。這就是查詢函數的威力!

📈 本週學習路徑

傳統函數 (VLOOKUP/HLOOKUP) ↓ 了解基本概念與限制 進階組合 (INDEX + MATCH) ↓ 突破傳統限制,更靈活 現代函數 (XLOOKUP) ↓ Excel 365 最佳解決方案 實務綜合應用 ↓ IE 與經濟學案例演練

2VLOOKUP 函數 - 傳統查詢利器

🔧 函數語法

=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup]) 參數說明: • lookup_value: 要查找的值(查詢鍵) • table_array: 查找範圍(必須包含查詢鍵和返回值) • col_index_num: 返回第幾欄的資料(從查找範圍的第1欄開始算) • range_lookup: TRUE=近似匹配 / FALSE=精確匹配(通常用FALSE)

📋 實務範例 1:產品成本查詢 (IE)

某電子零件製造商需要根據產品編號快速查找標準成本:

產品主檔表 (A1:D6) 訂單查詢表 (F1:H4)
產品編號 產品名稱 標準成本 庫存量 訂單產品 查詢成本 查詢庫存
IC-2001 微控制器 125.50 1,500 IC-2003 (公式) (公式)
IC-2002 記憶體晶片 89.20 2,300 IC-2001 (公式) (公式)
IC-2003 電源管理IC 156.80 800 IC-2005 (公式) (公式)
IC-2004 感測器晶片 210.00 650
IC-2005 通訊晶片 178.50 1,100

📝 公式範例

在 G2 儲存格輸入(查詢成本): =VLOOKUP(F2,$A$2:$D$6,3,FALSE) 在 H2 儲存格輸入(查詢庫存): =VLOOKUP(F2,$A$2:$D$6,4,FALSE) 公式說明: • F2: 查找"IC-2003" • $A$2:$D$6: 在產品主檔表中查找(使用絕對參照) • 3 或 4: 返回第3欄(標準成本)或第4欄(庫存量) • FALSE: 精確匹配產品編號
💡 實務技巧:
• 查找範圍使用絕對參照 ($A$2:$D$6) 方便公式向下複製
• col_index_num 是從查找範圍的第1欄開始計數,不是工作表的欄號
• 務必使用 FALSE 進行精確匹配,避免錯誤結果
• 查找值(產品編號)必須在查找範圍的第一欄

🌍 實務範例 2:經濟指標查詢 (Econ)

經濟學研究中經常需要根據國家代碼查找各項經濟指標:

國家經濟數據表 (2024年)
國家代碼 國家名稱 GDP (十億美元) 失業率 (%) 通膨率 (%)
USA 美國 27,974 3.8 3.2
CHN 中國 17,963 5.2 0.5
JPN 日本 4,231 2.6 2.8
DEU 德國 4,456 3.1 5.9
GBR 英國 3,495 4.2 4.0
查詢公式範例: =VLOOKUP("CHN",A2:E6,3,FALSE) → 返回中國的GDP: 17,963 =VLOOKUP("JPN",A2:E6,4,FALSE) → 返回日本的失業率: 2.6 =VLOOKUP("DEU",A2:E6,5,FALSE) → 返回德國的通膨率: 5.9

⚠️ VLOOKUP 的限制

✅ 優點

  • 語法相對簡單,容易學習
  • 適合簡單的垂直查找
  • 所有 Excel 版本都支援
  • 執行速度快

❌ 限制

  • 只能向右查找,無法向左
  • 查詢鍵必須在第一欄
  • 插入/刪除欄位會破壞公式
  • col_index_num 不直觀

🔍 範圍查詢:range_lookup = TRUE 的實務應用

當 range_lookup 設為 TRUE 時,VLOOKUP 會進行「近似匹配」,找到「小於等於查找值的最大值」。這在處理區間數據時非常有用。

⚠️ 使用 TRUE 的重要前提:
查找範圍的第一欄必須按升序排列,否則會返回錯誤結果!

📊 實務範例 1:運費級距計算 (IE/物流)

運費級距表 (A1:C6)
重量下限(kg) 運費(元) 說明
0 100 0-5kg
5 150 5-10kg
10 200 10-20kg
20 350 20-50kg
50 600 50kg以上
計算不同重量的運費: =VLOOKUP(3, A2:B6, 2, TRUE) → 100元 (3kg落在0-5kg區間) =VLOOKUP(15, A2:B6, 2, TRUE) → 200元 (15kg落在10-20kg區間) =VLOOKUP(45, A2:B6, 2, TRUE) → 350元 (45kg落在20-50kg區間) =VLOOKUP(80, A2:B6, 2, TRUE) → 600元 (80kg超過50kg) 運作原理: - VLOOKUP找到「小於等於查找值的最大值」 - 例如查找15kg時,找到的是10(≤15的最大值) - 然後返回對應的運費200元

💰 實務範例 2:所得稅率計算 (Econ/財務)

個人所得稅級距表 (A1:D6)
所得下限 所得上限 稅率(%) 累進差額
0 560,000 5 0
560,001 1,260,000 12 39,200
1,260,001 2,520,000 20 140,000
2,520,001 4,720,000 30 392,000
4,720,001 以上 40 864,000
查詢適用稅率: 年所得 800,000 元: =VLOOKUP(800000, A2:C6, 3, TRUE) → 12% (落在560,001-1,260,000區間) 年所得 1,800,000 元: =VLOOKUP(1800000, A2:C6, 3, TRUE) → 20% (落在1,260,001-2,520,000區間) 年所得 5,000,000 元: =VLOOKUP(5000000, A2:C6, 3, TRUE) → 40% (超過4,720,001) 完整稅額計算公式: =查詢所得*VLOOKUP(查詢所得,A2:C6,3,TRUE)/100-VLOOKUP(查詢所得,A2:D6,4,TRUE) ↑ ↑ 所得×稅率 累進差額

🎯 實務範例 3:學生成績等第判定 (教育)

成績等第對照表

0 F 不及格
60 D 及格
70 C 普通
80 B 良好
90 A 優秀
判定成績等第: =VLOOKUP(75, A2:B6, 2, TRUE) → "C" (75分落在70-80區間) =VLOOKUP(88, A2:B6, 2, TRUE) → "B" (88分落在80-90區間) =VLOOKUP(95, A2:B6, 2, TRUE) → "A" (95分超過90分) =VLOOKUP(55, A2:B6, 2, TRUE) → "F" (55分未達60分)
💡 TRUE vs FALSE 使用時機:

使用 TRUE (近似匹配) 當:
✅ 處理區間數據(運費級距、稅率級距、折扣級距)
✅ 分數評級、績效評等
✅ 數據已按升序排列
✅ 需要找到「小於等於」的匹配值

使用 FALSE (精確匹配) 當:
✅ 查找唯一標識符(產品編號、員工編號、訂單號)
✅ 需要完全相同的匹配
✅ 數據沒有排序或無法排序
✅ 大多數一般查詢情況 ← 預設建議
⚠️ 使用 TRUE 時的常見錯誤:

錯誤1:數據未排序
如果第一欄沒有按升序排列,會得到錯誤結果!
解決:使用前先對數據排序,或使用 FALSE + IF 組合

錯誤2:邊界值處理不當
要注意區間的上下限是否包含在內
解決:明確定義每個區間的範圍

錯誤3:混淆 TRUE 和 FALSE
大多數情況應該用 FALSE,只有區間查詢才用 TRUE
解決:不確定時優先使用 FALSE
🚫 常見錯誤與解決方案:

#N/A 錯誤: 找不到查找值
→ 檢查查找值是否存在於查找範圍的第一欄
→ 檢查是否有多餘空格或格式不一致

#REF! 錯誤: col_index_num 超出範圍
→ 確認欄索引號碼在查找範圍的欄數內

#VALUE! 錯誤: col_index_num 不是數字
→ 確保欄索引號碼是正整數

3HLOOKUP 函數 - 水平查詢

🔧 函數語法

=HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup]) 參數說明: • lookup_value: 要查找的值 • table_array: 查找範圍(水平排列) • row_index_num: 返回第幾列的資料 • range_lookup: TRUE=近似匹配 / FALSE=精確匹配

📊 實務範例:季度業績報表

項目 Q1 Q2 Q3 Q4
營收(百萬) 125.5 138.2 142.8 156.3
成本(百萬) 87.2 92.1 95.8 102.5
毛利率(%) 30.5 33.4 32.9 34.4
查詢公式範例: =HLOOKUP("Q3",A1:E1,2,FALSE) → 返回Q3的營收: 142.8 =HLOOKUP("Q2",A1:E1,4,FALSE) → 返回Q2的毛利率: 33.4
⚠️ HLOOKUP 使用建議:
HLOOKUP 較少使用,因為大多數數據表都是垂直排列。在實務上,如果遇到水平排列的數據,建議考慮:
• 將數據轉置為垂直排列,使用 VLOOKUP
• 使用 INDEX + MATCH 組合(更靈活)
• 使用 XLOOKUP(如果可用)

4INDEX + MATCH 組合 - 靈活的查詢方案

INDEX 和 MATCH 的組合是比 VLOOKUP 更強大、更靈活的查詢方式,可以突破 VLOOKUP 的所有限制。

🔧 函數拆解

📍 MATCH 函數 - 找位置

=MATCH(lookup_value, lookup_array, [match_type]) 作用: 返回查找值在範圍中的相對位置(第幾個) 參數說明: • lookup_value: 要查找的值 • lookup_array: 查找範圍(單行或單列) • match_type: 0=精確匹配(最常用) 1=小於等於(需升序排列) -1=大於等於(需降序排列) 範例: =MATCH("IC-2003",A2:A6,0) → 返回 3 (在第3個位置)

📍 INDEX 函數 - 取值

=INDEX(array, row_num, [column_num]) 作用: 返回範圍中指定位置的值 參數說明: • array: 查找範圍 • row_num: 第幾列 • column_num: 第幾欄(可選,單欄範圍時不需要) 範例: =INDEX(C2:C6,3) → 返回C2:C6範圍中第3個值

🎯 INDEX + MATCH 組合邏輯

=INDEX(返回值範圍, MATCH(查找值, 查找範圍, 0)) ↑ ↑ 取第N個值 找出位置N 執行流程: 1. MATCH 先找出查找值在查找範圍中的位置(例如:第3個) 2. INDEX 再根據這個位置,從返回值範圍中取出對應的值

📋 實務範例 1:向左查詢 (突破VLOOKUP限制)

VLOOKUP 無法做到的向左查詢,INDEX+MATCH 輕鬆解決:

產品名稱 產品編號 單價
微控制器 IC-2001 125.50
記憶體晶片 IC-2002 89.20
電源管理IC IC-2003 156.80
任務:根據產品編號(B欄)查找產品名稱(A欄) ← 向左查詢! VLOOKUP 無法做到(因為查詢鍵不在第一欄) INDEX + MATCH 解決方案: =INDEX(A2:A4, MATCH("IC-2003",B2:B4,0)) ↑ ↑ 產品名稱範圍 在產品編號中找位置 結果: "電源管理IC" 執行過程: 1. MATCH("IC-2003",B2:B4,0) → 返回 3 2. INDEX(A2:A4, 3) → 返回 A2:A4 中第3個值 → "電源管理IC"

📋 實務範例 2:IE 成本分析

完整產品成本資料表
產品編號 產品名稱 材料成本 人工成本 製造費用 總成本
P-101 齒輪組件 45.00 25.00 15.00 85.00
P-102 軸承模組 62.50 32.00 18.50 113.00
P-103 馬達組件 128.00 45.00 28.00 201.00
P-104 控制面板 95.00 38.00 22.00 155.00
查詢不同成本類型的公式: 查詢材料成本: =INDEX(C2:C5, MATCH("P-103",A2:A5,0)) → 128.00 查詢人工成本: =INDEX(D2:D5, MATCH("P-102",A2:A5,0)) → 32.00 查詢製造費用: =INDEX(E2:E5, MATCH("P-104",A2:A5,0)) → 22.00 優勢: - 可以查詢任何欄位,不受位置限制 - 插入/刪除欄位不會影響公式 - 邏輯清晰,易於維護

📊 實務範例 3:雙向查詢 (進階應用)

同時根據行和列條件查詢數據:

產品 北區 中區 南區 東區
產品A 125 138 142 156
產品B 87 92 95 102
產品C 210 225 218 234
任務:查詢"產品B"在"南區"的銷售量 =INDEX(B2:E4, MATCH("產品B",A2:A4,0), MATCH("南區",B1:E1,0)) ↑ ↑ ↑ 數據範圍 找出在第幾行 找出在第幾欄 執行過程: 1. MATCH("產品B",A2:A4,0) → 返回 2 (第2行) 2. MATCH("南區",B1:E1,0) → 返回 3 (第3欄) 3. INDEX(B2:E4, 2, 3) → 返回第2行第3欄的值 → 95
✅ INDEX + MATCH 優勢總結:
1. 方向自由: 可以向左、向右、向上、向下查詢
2. 不怕變動: 插入/刪除欄位不影響公式
3. 邏輯清晰: 查找位置和返回值分離,易於理解
4. 功能強大: 支持雙向查詢、多條件查詢
5. 效能優秀: 大數據量時比VLOOKUP更快

5XLOOKUP 函數 - 現代化查詢方案

⚠️ 版本需求:
XLOOKUP 僅在 Excel 365 和 Excel 2021 以後的版本中可用。如果你使用的是較舊版本,請使用 INDEX + MATCH 組合。

🔧 函數語法

=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode]) 參數說明: - lookup_value: 要查找的值 - lookup_array: 查找範圍 - return_array: 返回值範圍 - if_not_found: 找不到時返回的值(可選) - match_mode: 0=精確匹配(預設), -1=精確或小於, 1=精確或大於, 2=通配符匹配 - search_mode: 1=從頭搜尋(預設), -1=從尾搜尋, 2=二分搜尋(升序), -2=二分搜尋(降序)

📋 實務範例 1:基本使用

使用先前的產品成本表: XLOOKUP 查詢: =XLOOKUP("IC-2003", A2:A6, C2:C6) ↑ ↑ ↑ 查找"IC-2003" 在A欄中找 返回C欄的值 結果: 156.80 相比 VLOOKUP 的優勢: - 不需要計算欄位索引號 - 查找欄和返回欄可以是任意位置 - 語法更簡潔直觀

🎯 實務範例 2:找不到時的預設值

=XLOOKUP("IC-9999", A2:A6, C2:C6, "查無此產品") ↑ 找不到時顯示這個訊息 如果 VLOOKUP 要實現同樣效果: =IFERROR(VLOOKUP("IC-9999",A2:D6,3,FALSE), "查無此產品") ↑ 需要額外使用 IFERROR 函數

📊 實務範例 3:查詢整個欄位

查詢整個產品資訊(多欄): =XLOOKUP("IC-2003", A2:A6, B2:D6) ↑ 返回多個欄位(產品名稱、成本、庫存) 結果會返回一個陣列: ["電源管理IC", 156.80, 800] 這在建立動態報表時非常有用!

🔍 實務範例 4:反向查詢(從下往上找)

當有重複值時,找最後一個出現的項目: =XLOOKUP("產品A", A2:A10, B2:B10, "找不到", 0, -1) ↑ search_mode=-1 從尾端開始搜尋 應用場景: - 查詢客戶最新的訂單 - 查詢員工最近的考績記錄 - 查詢產品最新的價格異動

📈 實務範例 5:區間查詢(經濟學應用)

所得稅級距表
所得下限 稅率(%) 累進差額
0 5 0
560,000 12 39,200
1,260,000 20 140,000
2,520,000 30 392,000
4,720,000 40 864,000
查詢年所得 1,800,000 元適用的稅率: =XLOOKUP(1800000, A2:A6, B2:B6, , -1) ↑ match_mode=-1 (精確或小於) 結果: 20% (因為 1,800,000 落在 1,260,000 ~ 2,520,000 區間) 說明: - match_mode=-1 會找到"小於等於查找值的最大值" - 這在稅率級距、運費級距、折扣級距等場景非常實用

✅ XLOOKUP 優點

  • 語法簡潔,最直觀易懂
  • 可向任何方向查詢
  • 內建錯誤處理機制
  • 支持返回多個欄位
  • 支持反向搜尋
  • 支持區間查詢
  • 效能優於 VLOOKUP

⚠️ 使用限制

  • 僅限 Excel 365/2021+
  • 較舊版本無法使用
  • 與他人共享檔案需確認版本
💡 函數選擇建議:
優先使用: XLOOKUP (如果版本支持)
次優選擇: INDEX + MATCH (兼容性最佳)
簡單情境: VLOOKUP (向右查詢且數據結構不變時)
避免使用: HLOOKUP (建議轉置數據改用 VLOOKUP 或其他方法)

6常見問題與進階技巧

❓ 常見問題解答

Q1: 為什麼查詢結果是 #N/A?

可能原因:

  • 查找值在查找範圍中不存在
  • 數據格式不一致(例如:數字vs文字)
  • 有多餘的空格或不可見字符
  • 查找範圍選擇錯誤

解決方法:

使用 IFERROR 處理錯誤: =IFERROR(VLOOKUP(...), "查無資料") 或使用 XLOOKUP 的 if_not_found 參數: =XLOOKUP(..., "查無資料") 檢查空格問題: =XLOOKUP(TRIM(A1), ...) ← 使用TRIM去除空格

Q2: 如何處理重複值問題?

當查找範圍中有重複值時,查詢函數只會返回第一個匹配的結果。

解決方案:

  • 使用 XLOOKUP 的 search_mode=-1 從後往前找
  • 建立輔助欄位組合唯一鍵(例如:編號+日期)
  • 使用進階篩選或 FILTER 函數找出所有匹配項

Q3: 大數據量時查詢很慢怎麼辦?

效能優化技巧:

  • 使用 XLOOKUP 或 INDEX+MATCH 代替 VLOOKUP

⚡揮發性與非揮發性函數解析

🤔 什麼是揮發性函數?

揮發性函數 (Volatile Function) 是指每次 Excel 重新計算時都會執行的函數,即使其相依的儲存格沒有改變。

非揮發性函數 (一般函數) 揮發性函數
只在輸入值改變時重新計算 每次計算循環都會重新計算
效能好,適合大量數據 效能較差,可能造成延遲
Excel 可追蹤相依性 Excel 無法完全追蹤相依性
範例: VLOOKUP, INDEX, MATCH 範例: NOW, RAND, INDIRECT

⚙️ 揮發性函數的運作原理

非揮發性函數範例: A1: 100 A2: 200 A3: =A1+A2 執行過程: 1. 首次計算: A3 = 300 2. 改變 Z99 儲存格 → A3 不重新計算 (因為 A1、A2 沒變) 3. 改變 A1 → A3 重新計算 ✓ ---------------------------------------- 揮發性函數範例: A1: 100 A2: =INDIRECT("A1") 執行過程: 1. 首次計算: A2 = 100 2. 改變 Z99 儲存格 → A2 重新計算! (即使 A1 沒變) 3. 按 F9 → A2 重新計算! 4. 開啟檔案 → A2 重新計算! 原因: Excel 無法透過文字字串 "A1" 追蹤相依性, 所以保險起見,每次都重新計算

函數 用途 為何揮發

TODAY() 取得當前日期 日期會改變

🧪 如何測試函數是否揮發?

測試步驟: 步驟 1: 建立測試公式 A1: =你要測試的函數 A2: =RAND() (已知的揮發性函數,用於對照) 步驟 2: 按 F9 (強制重新計算) 觀察公式是否重新計算 步驟 3: 修改不相關的儲存格 在 Z99 輸入任何值 觀察 A1 是否重新計算 步驟 4: 判斷結果 ✓ 如果 A1 重新計算 → 揮發性函數 ✓ 如果 A1 不重新計算 → 非揮發性函數

⚡ 效能影響分析

🐌 揮發性函數對效能的影響:

情境: 工作簿包含 10,000 個公式

案例 A: 使用 100 個 INDIRECT 函數
• 每次任何儲存格改變,這 100 個公式都會重新計算
• 按 F9 時,這 100 個公式都會重新計算
• 開啟檔案時,這 100 個公式都會重新計算
• 結果: 明顯延遲,尤其是頻繁更新時

案例 B: 使用 100 個 INDEX+MATCH 組合
• 只有當相依儲存格改變時才重新計算
• 按 F9 時不會無謂重新計算
• 結果: 快速且有效率

🎯 最佳實務建議

✅ 優先使用非揮發性替代方案:

❌ 避免使用 ✅ 建議替代 說明
INDIRECT("A1") INDEX(A:A,1) 固定參照改用 INDEX
OFFSET(A1,0,0,5,1) A1:A5 固定範圍直接參照
INDIRECT(A1&"!B2") 使用命名範圍 跨工作表參照改用名稱
OFFSET(...) INDEX(..., MATCH(...)) 動態範圍改用 INDEX+MATCH
💡 何時可以接受使用揮發性函數?

可接受的情境:
✓ 小型工作簿(少於 1,000 個公式)
✓ 確實需要即時更新的資料(如 NOW, TODAY)
✓ 需要隨機數產生(如 RAND, RANDBETWEEN)
✓ 複雜的動態範圍且無其他解決方案
✓ 一次性使用,完成後轉為值

應該避免的情境:
✗ 大型數據分析工作簿
✗ 需要頻繁編輯的檔案
✗ 多人共享的檔案
✗ 有非揮發性替代方案時

🔧 效能優化實戰技巧

技巧 1: 將揮發性函數結果轉為靜態值 如果不需要即時更新: 1. 選取含有揮發性函數的儲存格 2. Ctrl+C (複製) 3. 在同一位置: Ctrl+Alt+V → 值 (將公式轉為值) 技巧 2: 限制揮發性函數的使用範圍 ❌ 不好: 在 10,000 行中都使用 INDIRECT ✅ 良好: 只在必要的幾個儲存格使用,其他參照該結果 技巧 3: 使用手動計算模式 當需要使用大量揮發性函數時: 公式 → 計算選項 → 手動 完成編輯後按 F9 手動重新計算 技巧 4: 使用 VBA 或 Power Query 預處理 對於複雜的動態參照需求: 考慮使用 VBA 或 Power Query 一次性處理 而非在儲存格中使用揮發性函數
