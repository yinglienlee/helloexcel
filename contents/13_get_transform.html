📥 資料匯入與清理

Get & Transform — 以 Power Query 整合、清洗與整形外部資料

1什麼是 Get & Transform？

Get & Transform 是 Excel 內建的資料整合功能，其核心引擎稱為 Power Query。它讓使用者能夠從各種外部來源匯入資料，並透過視覺化操作進行清洗、轉換和整形，全程不需撰寫 VBA 程式碼。

💡 專有名詞解釋：Power Query
Power Query 是 Microsoft 開發的資料連接與轉換技術，內建於 Excel 2016 及以後版本（舊版可安裝增益集）。每一個匯入與轉換步驟都會被記錄為「查詢步驟 (Applied Steps)」，日後只需按「重新整理 (Refresh)」即可自動更新資料，大幅提升工作效率。

如何開啟 Power Query

在 Excel 中，點擊上方功能列的 資料 (Data) 分頁，即可看到「取得及轉換資料」群組，其中包含「取得資料 (Get Data)」按鈕。

2外部資料匯入：CSV、Web、資料庫

Power Query 支援多種外部資料來源。以下介紹最常見的三種：

📄

CSV 檔案

最常見的純文字表格格式。路徑：
資料 → 取得資料 → 從檔案 → 從文字/CSV

🌐

Web 網頁

從網頁 URL 抓取表格資料。路徑：
資料 → 取得資料 → 從其他來源 → 從 Web

🗄️

資料庫

連接 SQL Server、Access 等資料庫系統。路徑：
資料 → 取得資料 → 從資料庫

以 CSV 為例：匯入步驟

步驟 1： 點擊「資料」→「取得資料」→「從檔案」→「從文字/CSV」
步驟 2： 瀏覽並選取目標 CSV 檔案，點擊「匯入」
步驟 3： 預覽視窗出現後，確認分隔符號（通常為逗號）與編碼格式（建議 UTF-8）是否正確
步驟 4： 點擊「轉換資料 (Transform Data)」進入 Power Query 編輯器，或直接點擊「載入 (Load)」匯入工作表
💡 專有名詞解釋：CSV (Comma-Separated Values)
CSV 是一種以純文字儲存表格資料的格式，每一列為一筆紀錄，欄位之間以逗號（,）分隔。由於格式簡單，幾乎所有資料系統（ERP、資料庫、政府開放資料平台）都支援匯出為 CSV 格式。
⚠️ 注意：中文編碼問題
若 CSV 檔案出現亂碼，通常是因為編碼格式不符。請在「檔案原始碼 (File Origin)」下拉選單中，嘗試選取 65001: Unicode (UTF-8) 或 950: 繁體中文 (Big5)。

3Power Query 基礎操作

3.1 資料類型轉換 (Change Data Type)

匯入的資料欄位不一定帶有正確的資料類型（例如數字被存為文字、日期格式不正確）。在 Power Query 中，可以透過以下方式修正：

  • 在欄位名稱旁的資料類型圖示（如 ABC、1.2、📅）上點擊，選擇正確類型。
  • 或在「常用」分頁中找到「資料類型」下拉選單進行設定。
資料類型 說明 常見應用
文字 (Text) 純文字，無法進行數學運算 姓名、國家代碼、備註
整數 (Whole Number) 不含小數點的數值 人數、件數、年份
小數 (Decimal Number) 含小數點的浮點數值 GDP、匯率、百分比
日期 (Date) 日期格式，可計算時間差 發布日期、交易日期
True/False 布林值，只有真或假 是否完成、是否有效
💡 為什麼資料類型這麼重要？
若欄位類型錯誤（如 GDP 數值被誤判為文字），後續的計算、排序、圖表將全部出錯。養成「匯入後立即檢查資料類型」的習慣，可避免許多下游分析錯誤。

3.2 行與列的轉置 (Transpose)

轉置是將資料表的行（列/橫向）與列（欄/縱向）互換的操作。當原始資料的欄位排列方向與分析需求不符時，轉置非常有用。

💡 專有名詞解釋：轉置 (Transpose)
轉置是指將資料矩陣的行與列互換。例如：原本「每欄代表一個年度」的格式，轉置後會變成「每列代表一個年度」。在 Power Query 中，路徑為：轉換 → 轉置。

範例：下表是原始格式（年份為欄標題）：

指標202120222023
GDP 成長率 (%)6.52.43.1
通膨率 (%)1.22.82.1

轉置後（年份成為列、指標成為欄），便於進行逐年趨勢分析：

年份GDP 成長率 (%)通膨率 (%)
20216.51.2
20222.42.8
20233.12.1

3.3 資料合併 (Merge) 與追加 (Append)

當資料分散在多個來源時，Power Query 提供兩種整合方式：

🔗 合併 (Merge)
橫向合併		 根據共同鍵值（Key），將兩個資料表以「欄」的方向合併。
類似 SQL 的 JOIN 或 Excel 的 VLOOKUP。
例如：將「國家代碼」作為鍵值，把 GDP 資料表與人口資料表合併成一張表。

路徑：常用 → 合併查詢 (Merge Queries)
📋 追加 (Append)
縱向合併		 將結構相同的資料表以「列」的方向堆疊在一起。
例如：將 2021、2022、2023 三年的月度 GDP 資料表（欄位相同）合併為一張完整的多年資料表。

路徑：常用 → 附加查詢 (Append Queries)
💡 Merge vs. Append：如何選擇？
一個簡單的判斷方式：
・如果你想增加更多欄位（新增屬性），使用 Merge（合併）。
・如果你想增加更多筆資料（新增紀錄），使用 Append（追加）。

Merge 的連結類型 (Join Kind)

連結類型 說明
Left Outer 保留左側資料表所有列，右側無對應資料則填 null。最常用。
Inner 只保留兩個資料表中「鍵值都存在」的列。
Full Outer 保留兩個資料表的所有列，互缺的欄位填 null。
Left Anti 只保留在左側表格有、但在右側表格中找不到對應鍵值的列。常用於找出「未匹配的資料」。

4資料清洗常用技巧

Power Query 提供豐富的清洗功能，以下是處理經濟數據時最常用的操作：

🚫 移除空白列

路徑：常用 → 移除列 → 移除空白列
消除匯入時產生的空白行，保持資料整潔。

✂️ 修剪 (Trim)

路徑：轉換 → 格式 → 修剪
移除欄位文字前後多餘的空白字元，防止比對失敗。

🔤 第一列做為標題

路徑：常用 → 使用第一列作為標題
當 CSV 第一列是欄位名稱時，使用此功能設為標題列。

📌 取代值 (Replace Values)

路徑：常用 → 取代值
將特定值（如空值、"N/A"）批次替換為正確內容或 null。

🔢 分割欄位 (Split Column)

路徑：常用 → 分割欄位
將一個欄位依分隔符號（如「/」、「-」）拆分成多欄。

🔃 移除重複值

路徑：常用 → 移除列 → 移除重複項目
針對選取的欄位，刪除重複紀錄。

💡 「套用的步驟 (Applied Steps)」面板
Power Query 編輯器右側的「套用的步驟」面板，記錄了所有操作。你可以點擊任一步驟返回該狀態，也可以刪除或重新排列步驟，讓整個清洗流程完全可控、可重現。

5實務範例：匯入多年度 GDP 數據並清洗

以下示範完整流程：從政府開放資料平台取得 2021–2023 年的年度 GDP 資料（CSV 格式），並使用 Power Query 進行清洗與格式化，最終產出一張可分析的整合表。

情境說明

你取得了三個 CSV 檔案，分別對應三個年度，每份格式相同，欄位如下：

國家代碼 國家名稱 年份 GDP（百萬美元） GDP 成長率（%） 人均 GDP（美元）
TWNTaiwan2023751,9301.3132,028
KORKorea, Rep.20231,712,7901.3633,147
JPNJapan20234,212,9501.9233,834
SGP Singapore2023497,3501.1084,500
N/A2023缺資料

清洗步驟示範（在 Power Query 中操作）

步驟 1：使用「追加 (Append)」合併三年資料
分別匯入 gdp_2021.csv、gdp_2022.csv、gdp_2023.csv 後，對其中一個查詢執行「附加查詢」，將三年資料縱向堆疊成一張表。
步驟 2：移除無效列
過濾掉「國家代碼」欄中值為 "N/A" 或空白的列，排除無效紀錄。
步驟 3：取代特殊值
將「GDP（百萬美元）」欄中的文字 "缺資料" 取代為 null，確保該欄位可被設為數字類型。
步驟 4：修正資料類型
將「年份」設為 整數，「GDP」與「人均 GDP」設為 小數，「國家代碼」設為 文字。
步驟 5：修剪國家名稱
對「國家名稱」欄執行「修剪」操作，移除 Singapore 欄位前的多餘空格。
步驟 6：載入至 Excel
點擊「常用 → 關閉並載入」，資料即匯入至一個新的工作表，並以 Excel 表格 (Table) 格式呈現。
【Power Query M 語言預覽（自動產生，不需手動輸入）】 let Source = Table.Combine({gdp_2021, gdp_2022, gdp_2023}), RemovedNullRows = Table.SelectRows(Source, each [國家代碼] <> "N/A" and [國家代碼] <> null), ReplacedErrors = Table.ReplaceValue(RemovedNullRows, "缺資料", null, Replacer.ReplaceValue, {"GDP（百萬美元）"}), ChangedTypes = Table.TransformColumnTypes(ReplacedErrors, { {"年份", Int64.Type}, {"GDP（百萬美元）", type number}, {"GDP 成長率（%）", type number} }), TrimmedNames = Table.TransformColumns(ChangedTypes, {{"國家名稱", Text.Trim}}) in TrimmedNames
💡 M 語言 (Power Query Formula Language)
Power Query 背後使用的腳本語言稱為 M 語言。雖然所有操作都可以透過 GUI 完成，但了解 M 語言語法有助於在進階情境下手動修改查詢邏輯。點擊「進階編輯器 (Advanced Editor)」可查看和編輯完整的 M 語言腳本。

✏️ 練習題

練習題 1：匯入並追加三年失業率 CSV 資料

任務描述：

主計總處提供了 2021、2022、2023 年三份 CSV 格式的各縣市失業率統計資料，格式完全相同。請使用 Power Query 將三份資料整合為一張完整的多年資料表，並進行基本清洗。

縣市代碼 (A 欄) 縣市名稱 (B 欄) 年份 (C 欄) 失業率 (%) (D 欄) 勞動力人口 (E 欄) 備註 (F 欄)
TPE台北市20213.451,289,000
NTC新北市20213.781,953,000
TCH台中市20213.211,415,000
KHH高雄市20213.911,366,000部分數據待確認
N/A2021格式異常列
💡 專有名詞解釋：
  • 失業率 (Unemployment Rate)： 失業人口占勞動力人口的百分比，是衡量經濟景氣的重要指標之一。
  • 勞動力人口 (Labor Force)： 年齡在勞動年齡範圍內、願意且能夠工作的人口總數，包含就業者與失業者。
  • 追加 (Append)： Power Query 中將多個欄位結構相同的表格縱向堆疊的操作，等同於 SQL 的 UNION ALL。

操作步驟與任務：

  1. 分別匯入 unemployment_2021.csv、unemployment_2022.csv、unemployment_2023.csv 三個查詢。
  2. 對第一個查詢執行「附加查詢 (Append Queries)」，選擇「三個以上的資料表」，將三年資料縱向合併為一張表（預期筆數 = 三年各縣市總和）。
  3. 過濾並移除「縣市代碼」為 "N/A" 或空白的無效列。
  4. 確認欄位資料類型：「年份」→ 整數、「失業率」→ 小數、「勞動力人口」→ 整數、其餘 → 文字。
  5. 對「縣市名稱」欄執行「修剪 (Trim)」，移除前後多餘空白。
  6. 將 "部分數據待確認" 的備註列保留，但在 Excel 中以條件式格式設定標記（選做）。
  7. 載入後，在 Excel 中確認：合併後的總列數是否正確（應為三年資料加總去掉無效列後的數量）。

練習題 2：合併 (Merge) GDP 資料與各國人口資料

任務描述：

你手邊有兩份資料：一份是 2023 年各國 GDP 資料，另一份是各國人口數量資料。請使用 Power Query 的「合併 (Merge)」功能，以「國家代碼」為共同鍵值，將人口數欄位併入 GDP 資料表，並計算各國的人均 GDP。

資料表 A：gdp_2023.csv（GDP 資料）

國家代碼 國家名稱 GDP（百萬美元）
TWNTaiwan751,930
KORKorea, Rep.1,712,790
JPNJapan4,212,950
THAThailand512,180

資料表 B：population_2023.csv（人口資料）

國家代碼 人口數（千人） 人口年增率（%）
TWN23,545-0.05
KOR51,7120.09
JPN124,352-0.53
VNM98,1860.88
💡 專有名詞解釋：
  • 合併 (Merge)： 在 Power Query 中，以「共同鍵值欄位」為依據，將兩張表的欄位橫向合併。類似 Excel 的 VLOOKUP，但更強大且支援多種 Join 類型。
  • Left Outer Join： 保留左側（主）資料表的所有列；右側若無對應鍵值，則該欄位填入 null。是最常見的 Merge 類型。
  • 人均 GDP (GDP per Capita)： 一國 GDP 除以人口數，常用來比較不同規模國家的生活水準或經濟發展程度。
  • 自訂資料行 (Custom Column)： Power Query 中可新增公式計算欄位的功能，路徑為「新增資料行 → 自訂資料行」。

操作步驟與任務：

  1. 分別匯入兩份 CSV，在 Power Query 中命名查詢為 GDP_2023 與 Population_2023。
  2. 在 GDP_2023 查詢中，執行「合併查詢 (Merge Queries)」，以「國家代碼」為鍵值，選擇 Population_2023 作為右側資料表，連結類型選 Left Outer。
  3. 展開合併後的欄位，僅勾選「人口數（千人）」加入主表（不保留重複的國家代碼欄位）。
  4. 確認 Thailand（THA）的人口欄位為 null（因為 Population_2023 中不含 THA，符合 Left Outer 行為）。
  5. 新增自訂資料行 (Custom Column)，命名為「人均 GDP（美元）」，公式為：
    = [GDP（百萬美元）] * 1000000 / ([人口數（千人）] * 1000)
  6. 將新欄位的資料類型設為小數 (Decimal Number)，並載入至 Excel。
  7. 在 Excel 中，確認 VNM（越南）的資料是否被包含（預期：不包含，因為 GDP 表中沒有 VNM）。

練習題 3：轉置寬格式經濟指標資料並進行格式化

任務描述：

世界銀行提供的部分開放資料採用「寬格式 (Wide Format)」，即每個年度為一欄，不利於 Excel 圖表繪製與長期趨勢分析。請使用 Power Query 將下列寬格式資料轉置為「長格式 (Long Format / Tall Format)」，並完成清洗與格式化。

原始資料（Wide Format，欄位為年度）：

指標代碼 指標名稱 國家代碼 2020 2021 2022 2023
NY.GDP.MKTP.KD.ZGGDP 成長率 (%)TWN3.366.532.431.31
FP.CPI.TOTL.ZG通貨膨脹率 (%)TWN-0.021.983.022.50
SL.UEM.TOTL.ZS失業率 (%)TWN3.853.953.673.47
NY.GDP.MKTP.KD.ZGGDP 成長率 (%)KOR-0.714.302.611.36
FP.CPI.TOTL.ZG通貨膨脹率 (%)KOR0.542.505.093.60

目標輸出格式（Long Format，每列代表一個年度觀測值）：

指標代碼 指標名稱 國家代碼 年份 數值
NY.GDP.MKTP.KD.ZGGDP 成長率 (%)TWN20203.36
NY.GDP.MKTP.KD.ZGGDP 成長率 (%)TWN20216.53
...............
💡 專有名詞解釋：
  • 寬格式 (Wide Format)： 每個觀察單位（如國家+指標）佔一列，不同時間點的數值分散在多個欄位（多年度欄）。此格式人眼易讀，但不適合統計分析與圖表。
  • 長格式 (Long Format / Tidy Data)： 每筆觀測值各佔一列，通常包含「識別欄」（如國家、指標）與「值欄」（如年份、數值）。此格式是統計分析、樞紐分析表與 BI 工具的標準輸入格式。
  • 取消資料行樞紐 (Unpivot Columns)： Power Query 中將「寬格式轉為長格式」的核心操作，路徑為「選取年度欄 → 轉換 → 取消其他資料行的樞紐 (Unpivot Other Columns)」。
  • 指標代碼 (Indicator Code)： 世界銀行等國際機構用來唯一識別統計指標的代碼，如 NY.GDP.MKTP.KD.ZG 代表以固定美元計算的 GDP 年增率。

操作步驟與任務：

  1. 將原始寬格式資料匯入 Power Query（可直接從 Excel 工作表建立查詢，或從 CSV 匯入）。
  2. 選取「指標代碼」、「指標名稱」、「國家代碼」三欄（按住 Ctrl 多選），執行「取消其他資料行的樞紐 (Unpivot Other Columns)」，將年度欄轉換為長格式。
  3. 將 Power Query 自動產生的「屬性 (Attribute)」欄重新命名為「年份」，「值 (Value)」欄重新命名為「數值」。
  4. 將「年份」欄的資料類型設為整數，「數值」欄設為小數。
  5. 確認最終列數：原始 5 列 × 4 個年度欄 = 20 列（加上標題列後共 21 列）。
  6. 載入至 Excel，並依「國家代碼」→「指標代碼」→「年份」的順序排序，確認資料整齊排列。
  7. 進階挑戰：利用整理後的長格式資料，建立一個「折線圖」，比較台灣（TWN）與韓國（KOR）的 GDP 成長率走勢（2020–2023）。

🎓 學習重點回顧

  • Power Query：Excel 內建的資料整合引擎，支援 CSV、Web、資料庫等多種來源，所有操作自動記錄為可重複執行的步驟。
  • 資料類型轉換：匯入後務必確認欄位類型，數值誤為文字將導致計算錯誤。
  • 轉置 (Transpose)：行與列互換，適用於原始格式不符分析需求的情境。
  • 合併 (Merge)：以共同鍵值欄位橫向合併兩表，類似 VLOOKUP；Left Outer 是最常用的連結類型。
  • 追加 (Append)：將欄位相同的多張表縱向堆疊，適用於多年度資料整合。
  • 寬格式 vs. 長格式：使用「取消資料行樞紐」將寬格式轉為長格式，是 Power Query 清洗經濟數據的核心技能。
