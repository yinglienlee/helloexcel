🎯 邏輯判斷函數教學

掌握條件判斷與商業決策邏輯

1IF 函數基礎

📖 專有名詞解釋:
• 邏輯判斷 (Logical Test):判斷某個條件是否成立的過程,結果為 TRUE (真) 或 FALSE (假)。

• 條件運算子 (Comparison Operators):用於比較數值或文字的符號,包括 = (等於)、> (大於)、< (小於)、>= (大於等於)、<= (小於等於)、<> (不等於)。

• 傳回值 (Return Value):當條件成立或不成立時,函數返回的結果。

📌 IF 函數語法

=IF(邏輯測試, 值_如果為真, 值_如果為假) =IF(logical_test, value_if_true, value_if_false)

參數說明:

  • 邏輯測試:要判斷的條件 (例如: A1>100)
  • 值_如果為真:條件成立時傳回的值
  • 值_如果為假:條件不成立時傳回的值

基本應用範例

範例 1: 判斷及格與否 =IF(A2>=60, "及格", "不及格") 解釋: 如果 A2 的分數大於等於 60,顯示"及格",否則顯示"不及格" 範例 2: 判斷是否達標 =IF(B2>=10000, "達標", "未達標") 解釋: 如果 B2 的銷售額大於等於 10000,顯示"達標",否則顯示"未達標" 範例 3: 計算獎金 =IF(C2>100000, C2*0.1, 0) 解釋: 如果 C2 業績超過 100000,獎金為業績的 10%,否則沒有獎金
⚠️ 常見錯誤:
  • 文字參數忘記加雙引號: =IF(A1>60, 及格, 不及格) ❌ 正確: =IF(A1>60, "及格", "不及格") ✅
  • 比較運算子使用錯誤: =IF(A1=<60...) ❌ 正確: =IF(A1<=60...) ✅
  • 括號不匹配: =IF(A1>60, "及格" ❌ 正確: =IF(A1>60, "及格", "不及格") ✅

2巢狀 IF 結構

📖 專有名詞解釋:
• 巢狀結構 (Nested Structure):在一個 IF 函數內再包含另一個 IF 函數,用於處理多重條件判斷。

• 多層判斷 (Multi-level Decision):根據不同條件範圍,返回不同結果的判斷邏輯,如成績分級、信用評級等。

巢狀 IF 語法結構

=IF(條件1, 結果1, IF(條件2, 結果2, IF(條件3, 結果3, 其他結果)))

邏輯流程圖:

第一層判斷: 條件1 成立? → 是 → 傳回結果1
第二層判斷: 條件1 不成立,條件2 成立? → 是 → 傳回結果2
第三層判斷: 條件2 不成立,條件3 成立? → 是 → 傳回結果3
最終結果: 所有條件都不成立 → 傳回其他結果

實務範例:成績分級

範例: 學生成績評級 =IF(A2>=90, "優秀", IF(A2>=80, "良好", IF(A2>=70, "中等", IF(A2>=60, "及格", "不及格")))) 判斷邏輯: • 90 分以上 → 優秀 • 80-89 分 → 良好 • 70-79 分 → 中等 • 60-69 分 → 及格 • 60 分以下 → 不及格

  • 判斷順序很重要:從最高條件開始判斷,否則邏輯會出錯
  • 括號要成對:每個 IF 都需要對應的右括號,建議邊寫邊數括號
  • 可讀性問題:超過 3-4 層巢狀會很難閱讀,建議改用 IFS 函數
  • Excel 限制:最多支援 64 層巢狀,但實務上很少超過 5 層

3邏輯運算函數: AND, OR, NOT

📖 專有名詞解釋:
• 邏輯運算 (Logical Operations):對多個條件進行組合判斷的運算方式。

• AND 邏輯 (邏輯且):所有條件都必須成立,結果才為 TRUE,只要有一個不成立就為 FALSE。

• OR 邏輯 (邏輯或):只要有任何一個條件成立,結果就為 TRUE,所有條件都不成立才為 FALSE。

• NOT 邏輯 (邏輯非):將條件結果反轉,TRUE 變 FALSE,FALSE 變 TRUE。

📌 AND 函數

=AND(條件1, 條件2, 條件3, ...) =AND(logical1, logical2, logical3, ...)

功能:判斷所有條件是否同時成立

傳回值:所有條件都為 TRUE 時傳回 TRUE,否則傳回 FALSE

範例應用:

範例 1: 判斷是否符合貸款條件 (年收入≥500,000 且 信用評分≥700) =IF(AND(A2>=500000, B2>=700), "核准", "拒絕") 範例 2: 判斷優秀學生 (出勤率≥95% 且 成績≥85) =IF(AND(C2>=0.95, D2>=85), "優秀學生", "") 範例 3: 多條件獎金判定 (業績>100萬 且 客戶滿意度>90% 且 無客訴) =IF(AND(E2>1000000, F2>0.9, G2=0), E2*0.05, 0)

📌 OR 函數

=OR(條件1, 條件2, 條件3, ...) =OR(logical1, logical2, logical3, ...)

功能:判斷是否至少有一個條件成立

傳回值:任一條件為 TRUE 時傳回 TRUE,全部為 FALSE 才傳回 FALSE

範例應用:

範例 1: 判斷是否需要補考 (數學<60 或 英文<60 或 國文<60) =IF(OR(A2<60, B2<60, C2<60), "需要補考", "通過") 範例 2: VIP 客戶判定 (年消費≥100萬 或 會員等級=鑽石 或 推薦客戶≥10) =IF(OR(D2>=1000000, E2="鑽石", F2>=10), "VIP", "一般會員") 範例 3: 緊急訂單判定 (標記為緊急 或 客戶為重要客戶 或 訂單金額>50萬) =IF(OR(G2="緊急", H2="重要", I2>500000), "優先處理", "正常處理")

📌 NOT 函數

=NOT(條件) =NOT(logical)

功能:將邏輯值反轉

傳回值:條件為 TRUE 時傳回 FALSE,條件為 FALSE 時傳回 TRUE

範例應用:

範例 1: 判斷不是 VIP 客戶 =IF(NOT(A2="VIP"), "一般客戶", "VIP客戶") 範例 2: 判斷未完成訂單 =IF(NOT(B2="已完成"), "待處理", "已完成") 範例 3: 組合使用 - 不是新客戶且消費金額>10000 =IF(AND(NOT(C2="新客戶"), D2>10000), "給予折扣", "")

邏輯函數組合應用

情境 邏輯條件 公式範例
同時滿足多個條件 條件A 且 條件B =IF(AND(A2>100, B2="已付款"), "符合", "不符合")
滿足任一條件 條件A 或 條件B =IF(OR(A2="VIP", B2>50000), "優惠", "原價")
複雜組合 (A且B) 或 C (A AND B) OR C =IF(OR(AND(A2>100, B2="是"), C2="緊急"), "Y", "N")
排除特定條件 不是 A =IF(NOT(A2="取消"), "有效訂單", "已取消")
A 或 B 但不是同時 (A OR B) AND NOT(A AND B) =IF(AND(OR(A2,B2), NOT(AND(A2,B2))), "是", "否")
💡 實務技巧:
  • 使用 AND 時,把最容易判斷為 FALSE 的條件放前面,可提升效能
  • 使用 OR 時,把最容易判斷為 TRUE 的條件放前面
  • 複雜邏輯可以用輔助欄位拆解,讓公式更易讀
  • 善用括號清楚表達邏輯層次

4IFS 函數 (Excel 2019+)

📖 專有名詞解釋:
• IFS 函數 (Multiple IF Function):Excel 2019 及更新版本提供的多條件判斷函數,用於取代複雜的巢狀 IF 結構。

• 條件-結果配對 (Condition-Result Pairs):IFS 函數以「條件,結果」的方式成對輸入,依序判斷直到找到第一個成立的條件。

• 向下相容性 (Backward Compatibility):舊版 Excel (2016 及更早版本) 不支援 IFS 函數,開啟檔案會顯示錯誤。

📌 IFS 函數語法

=IFS(條件1, 結果1, 條件2, 結果2, 條件3, 結果3, ...) =IFS(logical_test1, value_if_true1, logical_test2, value_if_true2, ...)

特點:

  • 可以處理多達 127 組條件-結果配對
  • 從左到右依序判斷,遇到第一個成立的條件就停止
  • 所有條件都不成立時會傳回 #N/A 錯誤
  • 比巢狀 IF 更簡潔易讀

IFS vs 巢狀 IF 比較

巢狀 IF 寫法
=IF(A2>=90, "優秀", IF(A2>=80, "良好", IF(A2>=70, "中等", IF(A2>=60, "及格", "不及格")))) • 需要多層括號 • 難以閱讀和維護 • 容易漏掉括號
IFS 寫法
=IFS(A2>=90, "優秀", A2>=80, "良好", A2>=70, "中等", A2>=60, "及格", A2<60, "不及格") • 結構清晰平鋪 • 容易閱讀和修改 • 條件明確對應結果

實務應用範例

範例 1: 訂單量折扣判定 =IFS(B2>=1000, 0.15, B2>=500, 0.10, B2>=200, 0.05, B2>=100, 0.02, B2<100, 0) 判斷邏輯: • 訂購量 ≥ 1000 → 15% 折扣 • 訂購量 ≥ 500 → 10% 折扣 • 訂購量 ≥ 200 → 5% 折扣 • 訂購量 ≥ 100 → 2% 折扣 • 訂購量 < 100 → 無折扣 範例 2: 客戶信用評級 =IFS(C2>=800, "AAA", C2>=750, "AA", C2>=700, "A", C2>=650, "BBB", C2>=600, "BB", C2<600, "B") 範例 3: 員工績效獎金比例 (結合 AND 函數) =IFS(AND(D2>=95, E2="無"), 0.08, AND(D2>=90, E2="無"), 0.06, AND(D2>=85, E2="無"), 0.04, D2>=80, 0.02, TRUE, 0) 說明: 績效分數≥95 且無違規 → 8% 獎金 績效分數≥90 且無違規 → 6% 獎金 績效分數≥85 且無違規 → 4% 獎金 績效分數≥80 (不論是否違規) → 2% 獎金 其他情況 → 無獎金
⚠️ IFS 函數重要提醒:
  • 預設值處理:建議最後加上 TRUE 條件作為預設值,避免 #N/A 錯誤
    範例: =IFS(A2>100, "高", A2>50, "中", TRUE, "低")
  • 條件順序:IFS 從左到右判斷,遇到第一個 TRUE 就停止,所以要按照從嚴格到寬鬆的順序
  • 相容性問題:Excel 2016 及更早版本不支援 IFS,需改用巢狀 IF
  • 錯誤偵錯:檢查是否有條件永遠無法被執行到 (被前面的條件覆蓋)
✅ 何時使用 IFS:
  • 需要判斷 3 個以上的條件時
  • 條件邏輯需要清晰易讀時
  • 團隊成員都使用 Excel 2019 或更新版本時
  • 需要頻繁修改條件邏輯時
❌ 何時不使用 IFS:
  • 需要與舊版 Excel 相容時 → 改用巢狀 IF
  • 只有 1-2 個簡單條件時 → 用基本 IF 就夠了
  • 條件過於複雜時 → 考慮用輔助欄位拆解

5實務案例 1: 訂單折扣判定系統

📖 商業情境說明:
• 數量折扣 (Volume Discount):根據訂購數量給予不同折扣比例,鼓勵大量採購。

• 客戶等級 (Customer Tier):依據客戶歷史交易記錄、信用狀況等評定等級,不同等級享有不同優惠。

6實務案例 2: 貸款信用評級系統

📖 金融術語解釋:
• 信用評分 (Credit Score):根據借款人的財務狀況、還款紀錄等因素計算的分數,用於評估違約風險。

• 負債比 (Debt-to-Income Ratio, DTI):每月負債支出佔月收入的比例,銀行用來評估還款能力。計算公式:月負債支出 ÷ 月收入。

• 貸款成數 (Loan-to-Value Ratio, LTV):貸款金額佔擔保品價值的比例,例如房屋價值 1000 萬,貸款 800 萬,LTV = 80%。

• 核貸利率 (Approved Interest Rate):根據借款人信用風險決定的貸款利率,風險越高利率越高。

第二階段:負債比必須 < 60%,否則拒絕

📖 專業術語補充說明（邏輯判斷函數實務應用）

  • 達成率 (Achievement Rate):實際績效相對於目標值的比率，通常以百分比表示。計算公式：實際值 ÷ 目標值 × 100%。
  • 安全庫存 (Safety Stock):為應對需求波動或供貨延遲而預留的最低庫存量，通常以「安全天數 × 日均銷量」計算。
  • 交期 (Lead Time):從下訂單到收到貨物的所需天數，是補貨決策的重要參數。
  • 在途庫存 (On-order / In-transit Inventory):已下訂單但尚未入庫的庫存量，需納入庫存天數計算以避免重複補貨。
  • 季節係數 (Seasonality Index):反映不同時期需求變動的調整係數，例如旺季 >1、淡季 <1，用於修正日均銷量預測。
  • 庫存天數 (Days of Inventory / DOI):目前庫存可支應未來銷售的天數，計算公式：(現有庫存 + 在途) ÷ 調整後日銷量。
  • 補貨點 (Reorder Point, ROP):當庫存降至此水平時需立即補貨的臨界值，通常為「交期需求 + 安全庫存」。
  • 經濟訂購量 (Economic Order Quantity, EOQ):綜合考量訂購成本與持有成本的最優單次訂購數量（本教學未涉及數學公式，但實務常見）。
  • 負債比 (Debt-to-Income Ratio, DTI):每月債務支出佔月收入的比例，用於評估還款能力，通常 <36% 為佳。
  • 貸款成數 (Loan-to-Value Ratio, LTV):貸款金額佔擔保品價值的比率，數值越低風險越低。
  • 複合折扣 (Compound Discount):多項折扣同時適用時的正確計算方式：1 - (1-折扣1) × (1-折扣2) × (1-折扣3)，避免單純相加導致過度折扣。
  • 加權平均 (Weighted Average):依不同權重計算綜合評分的常見方法，例如品質40%、交期30%、價格20%、配合度10%。
  • 邊界條件 (Edge Case):公式測試時需特別驗證的極端情況，例如分母為0、最大/最小值、空白儲存格等。
  • 防呆設計 (Error-proofing / Poka-Yoke):在公式中加入 IFERROR、IF(Divisor=0,...) 等判斷，避免 #DIV/0!、#N/A 等錯誤影響後續計算。

計算規則:

    • 庫存天數 > 安全天數 + 供貨商交期 + 14 → "過剩"

  • 建議補貨量 (K欄):

    • 庫存狀態「警示」→ "中"
    • 其他 → "低"

  • 實務應用：大幅提升篩選效率，確保不錯過優秀人才
  • 商業價值：優化人才獲取成本與品質，支援快速擴員需求

🎓 學習重點回顧

  • IF 函數:單一條件判斷的基礎,語法 =IF(條件, 真值, 假值)
  • 巢狀 IF:處理多層次條件,但超過3-4層建議改用 IFS
  • AND 函數:所有條件都要成立,適合「且」的邏輯
  • OR 函數:任一條件成立即可,適合「或」的邏輯
  • NOT 函數:反轉邏輯結果,常與 AND/OR 組合使用
  • IFS 函數:Excel 2019+ 的多條件判斷利器,記得加 TRUE 作為預設值
  • 組合應用:實務中常需要 IF+AND+OR 的組合,要注意括號配對和邏輯順序
  • 除錯技巧:複雜公式可用輔助欄位拆解,逐步驗證每個判斷條件

💼 商業應用場景

  • 金融業:貸款審核、信用評級、風險評估、投資組合配置
  • 零售業:會員分級、折扣計算、庫存預警、訂單優先級
  • 製造業:品質判定、供應商評鑑、生產排程、設備維護提醒
  • 人資管理:績效考核、薪資計算、招聘篩選、培訓需求評估
  • 教育領域:成績評級、獎學金評定、課程推薦、學習預警

⚠️ 實務注意事項

  • 判斷順序:條件從嚴格到寬鬆排列,避免邏輯錯誤
  • 預設值:IFS 函數最後加上 TRUE 條件處理例外情況
  • 可讀性:複雜邏輯用輔助欄位拆解,增加公式可維護性
  • 測試驗證:準備各種測試案例,確保邊界條件正確
  • 文檔記錄:在儲存格註解或工作表說明欄記錄判斷邏輯
  • 相容性:IFS 函數不支援舊版 Excel,需考慮使用者環境
